§1.52 Class NoClassDefFoundError

The Java runtime throws an instance of this class if the runtime tries to load in the definition of a class (as part of a normal method call or as part of creating a new instance using the new expression) and no definition of the class could be found.

The searched-for class definition existed when the currently executing class was compiled, but the definition can no longer be found.
